# 위치 기반 관광지 추천 대시보드 실행 계획서 (Location-based Tourist Attraction Dashboard)

## 1. 개요 (Overview)
- **목표:** `tour_rag/data` 폴더의 데이터를 활용하여 사용자 위치, 연령대, 관심사(SNS 핫플, K-콘텐츠 촬영지 등)를 기반으로 맞춤형 관광지를 추천하는 интерактив 웹 대시보드(Streamlit 권장) 구축.
- **주요 타겟:** 스마트 관광을 원하는 내/외국인 여행객 및 특정 테마(K-컬처, 인증샷 등)에 관심 있는 세대별 사용자.

---

## 2. 데이터 활용 방안 (Data Pipeline & Utilization)

대시보드의 추천 정확도를 높이기 위해 제공된 데이터를 다음과 같이 결합하여 사용합니다.

### A. 세대별 / 트렌드 기반 기본 추천
- **대상 파일:** `세대별 인기관광지.csv`, `세대별 핫플레이스.csv`, `방문자 급등동네.csv`
- **활용:** 
  - 사용자가 '연령대'를 선택하면 해당 연령에서 비율/성장율이 높은 핫플레이스를 우선 추천.
  - 지도상에 급상승 중인 동네를 히트맵(Heatmap) 형태로 시각화.

### B. 미디어콘텐츠 & K-컬처 테마 추천
- **대상 파일:** `EC_SNS_KMOVIE_ATTRACTION_DATA_*.csv`, `영화 로케이션 촬영 이력.csv`, `미디어콘텐츠 영상 촬영지 데이터.csv`, `K-컬처 키워드*.csv`
- **활용:**
  - 영화/드라마 촬영지 필터를 적용하여 성지순례용 K-컬처 관광지 추천.
  - SNS 채널(블로그, 인스타 등)에서 언급량이 높은 영화 로케이션의 위/경도(`LC_LA`, `LC_LO`)를 활용해 지도상에 마커 표시.
  - 실시간 K-컬처 트렌드 랭킹에 맞춘 추천 알고리즘 가중치 부여.

## 3. 대시보드 주요 기능 및 아키텍처 (Dashboard Architecture)

### 3.1. 화면 구성 (UI/UX)
1. **사이드바 (Sidebar): 사용자 프로파일링 및 필터링**
   - **현재 위치(가상 주소/좌표 입력):** 반경 N km 이내 추천 검색 반경 설정 기능
   - **연령대 필터:** 20대, 30대, ... 전체
   - **테마 필터:** K-Movie 촬영지, 핫플레이스, 급상승 지역, 인증샷 성지(SNS 반응도 기준)

2. **메인 화면 (Main View)**
   - **KPI 요약 카드:** 선택된 연령대/테마의 평균 성장률, SNS 최다 조회수 관광지 요약표.
   - **위치 기반 추천 지도 (Map Interface):** `Folium` 또는 `PyDeck`을 이용한 클러스터 마커 매핑.
     - 마커 툴팁: 관광지명, 평균 평점(`AVRG_SCORE_VALUE`), SNS 리뷰 요약.
   - **Top 10 추천 리스트 (데이터 테이블/카드):** 이미지나 카드 레이아웃으로 상세 정보 및 추천 근거 시각화.

### 3.2. RAG(Retrieval-Augmented Generation) 챗봇 연동 (선택사항)
- `tour_rag` 디렉토리 목적에 부합하게, 관광지 관련 텍스트 메타데이터(리뷰 요약, 영화 정보, 위치 묘사 등)를 벡터화(FAISS/Chroma 등)하여 추천 질문 챗봇을 대시보드에 내장.
- **예시 질문:** "지금 위치(강남)에서 20대가 많이 가는 영화 촬영지 추천해 줘."

## 4. 단계별 실행 계획 (Implementation Steps)

### Phase 1: 데이터 전처리 및 병합 (Pre-processing)
- `EC_SNS_KMOVIE` 데이터 월별 취합 및 위경도 결측치 정제.
- `영화 로케이션` 데이터와 `관광명소/핫플레이스` 데이터간 교집합 매핑(이름, 위치 기준).
- RAG를 위한 관광지별 소개 텍스트 및 메타데이터 문서화(Document Chunking).

### Phase 2: 핵심 백엔드/로직 구현 (Backend & Logic)
- **공간 쿼리 알고리즘:** 사용자 위치 기반 반경 검색 (Haversine 공식 또는 Geopandas 활용).
- **RAG 파이프라인 구성:** LangChain / LlamaIndex를 활용해 텍스트 데이터의 Vector DB 저장.
- **추천 스코어링:** (거리 + 연령별 인기도 + SNS 리뷰수/평점) 복합 가중치 적용.

### Phase 3: 프론트엔드 대시보드 개발 (Streamlit)
- UI 레이아웃 퍼블리싱 (Sidebar, Map, DataFrame).
- 데이터 변화에 원활한 반응을 위한 렌더링 최적화 (`@st.cache_data` 적용).
- `koreanize-matplotlib` 등 한글 폰트 적용 시각화 차트 추가(성장률 그래프 등).

### Phase 4: 테스트 및 고도화 (Refinement)
- 가상의 좌표를 짚어가며 거리/테마 필터 정상 동작 확인.
- UI 직관성 개선 및 챗봇 연동 최종 점검.
